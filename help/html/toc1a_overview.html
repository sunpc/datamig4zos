Process overview

The tool supports to migrate both DB2 tables and datasets from one z/OS server to another, and runs a query in batch mode.

Table migration process

The table migration process includes unload, FTP, and load steps.

  • Allocates a control library on both source and target servers.
  • Submits an unload job to the source server to unload the data from the source table.
  • Once the unload is done, submits an FTP job to the target server to transmit the unloaded dataset.
  • Once the FTP is done, downloads the unload control card and use it to generate a load card.
  • Uploads the load card to the target server.
  • Once the upload is done, submits a load job to the target server to load the data into the target table.
  • Once the load is done, the process is complete.

    • Dataset migration process

    The dataset migration process includes an FTP step.

  • Allocates a control library on both source and target servers.
  • Submits an FTP job to the target server to transmit the dataset.
  • Once the FTP is done, the process is complete.

    • Database query process

    The database query process includes query and download steps.

  • Allocates a control library on the source server.
  • Submits a query job to the server to run the query.
  • Once the query is done, downloads the query results.
  • Displays the query results.
